Full-Stack App: DeepMBTI

DeepMBTI is an AI-powered MBTI full-stack predictor application. The app includes an interactive front-end that links to an NLP Log Reg ML Model back-end, utilizing HTML/CSS, Python, Flask, PostgreSQL, and Sklearn. DeepMBTI was the capstone project for Columbia University Data Analytics and Visualization Bootcamp.

Problem

Myers-Briggs Type Indicator (MBTI) is one of the most prevalent personality tests in psychology. Its purpose is to classify psychological profiles based on the assumption that variations in everyday behavior are ordered, consistent and capable of being categorized. Currently, the MBTI test is performed by an online quiz. However, these tests often include intrinsic biases because the subject is conscious of the quiz's processes and will answer accordingly.

Motivation

Therefore, we wanted to create a more objective process for testing MBTI. By inputting writing from the subject, we hope to train a machine learning model to predict their personality category through text analytics. Hopefully, this will eliminate input biases from the subject.

Aims

- Explore and analyze the data to see if any patterns can be detected in specific personality types and their writing styles.
- Create a Machine Learning model capable of predicting personality type based on the writing input, so the test taker bias is removed.

Processes

The development of DeepMBTI took on three crucial steps. These broad three steps includes examining the data and extracting important, selecting the most accurate ML model, and developing an interactive application

01.

Data Analysis

The dataset contains ~ 8,600 observations (people) imported from Kaggle, where each observation gives a person’s: Myers-Briggs personality type (as a four-letter code). An excerpt containing the last 50 posts on their PersonalityCafe forum (each entry separated by “|||”). After an EDA, we resampled our data since some of the dimensions were not balanced.

02.

Machine Learning Models

For our ML models, we chose Logistic Regression as our final ML model. That said, we tested four different ML models to see which one would provide us the best F1 accuracy. These models include: Log Reg, SVM, Random Forst, and Neural Network

03.

Interactive Application

The weights of our model and a vectorizor was saved using pickle. It was then imported infor Flask and connected to its front-end.

Technologies Used

DeepMBTI's full-stack design utilizes an NLP Logistic Regression ML Model back-end, HTML/CSS, Python, Flask, PostgreSQL, and Sklearn.

Results

The front-end allows the user to input some of their writings or recent posts on social media. Future Developement

  • Find a better dataset that fits with our purpose of the front-end application: Most of the visitors to our page will likely type some sentences right away, or copy some of their own writings from school or work.However, the dataset we used to train our models primarily consisted of posts in the forum or on social media. The content and style of the writing may be very different

  • We can make our front-end more appealing in terms of content, styling, and creative elements. In the results section, we can create individual explanations for each of the 16 MBTI Types.

  • We can integrate this model into Twilio or Chrome Plugins, so that it could analyze user personality as they work: Commercialization
